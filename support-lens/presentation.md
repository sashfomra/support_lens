## 3️⃣ 📋 Ticket Detail Modal

Click any ticket row to open the full detail panel.

### Panels inside the modal

#### Header
- Urgency badge + score, emotion badge, Churn Risk ⚠️ flag, Escalated 🔺 flag
- Source tag (voice / web / reddit / etc.), Customer tier

#### AI 3-Line Summary
- Numbered ①②③ concise points generated by `llama3.2:3b`
- Gives agent instant context without reading full message

#### Customer Message
- Full original text shown (PII is masked in the AI-processed version)

#### 🔄 Customer 360 Context
- Pulls **all previous tickets** from the same customer email
- Shows sentiment trend as an emoji timeline across tickets
- Detects **"High Friction Account"** badge if avg emotion intensity is high
- Shows ticket count and average urgency of past interactions
- Helps agent understand if this is a repeat complainer or first-time issue

#### 🔍 Find Solution (AI Solution Engine)
**Flow:**
1. Agent clicks **"Find Solution"**
2. System runs semantic search across 3 sources simultaneously:
   - **ChromaDB** — internal knowledge base articles (RAG)
   - **Reddit** — scraped complaint threads
   - **Stack Overflow** — technical solutions
3. Returns top matches with **confidence score** (%)
4. If confidence ≥ 75% → marked as "Verified KB" solution
5. If confidence < 75% → marked as "General AI Knowledge" with a ⚠️ review warning
6. Each result shows **source chips** (clickable links to original doc/thread)
7. Agent can click **"Use as Draft"** → injects solution into the reply box
8. Agent can rate each solution **👍 or 👎** → feedback stored for ranking improvement

#### ✉️ AI Draft Reply
- Generated by `llama3.2:3b` with tone adapted to emotion + tier:
  - Angry enterprise customer → empathetic + escalation-ready tone
  - Confused standard user → simple, instructional tone
- **Policy Filter** runs automatically:
  - Strips refund promises, legal guarantees, off-brand language
  - Shows red flag badges for any violations
  - Confidence score degrades if policy flags triggered
- **"Open in Mail Client"** button → opens native email app with pre-filled To, Subject, and body
- **"Save & Mark In Progress"** — updates ticket status in DB

## 4️⃣ 📥 Submit Ticket — AI Pipeline Demo

**URL:** `/inbox`

### Voice Note Transcription
1. Agent uploads audio file (`.mp3 .wav .ogg .m4a .webm .flac`, up to 25MB)
2. Backend receives file → saves as temp file
3. **ffmpeg** converts audio to 16kHz mono WAV
4. **OpenAI Whisper `base`** transcribes the WAV to text (runs 100% locally, no API key)
5. Transcript sent to **Groq LLM** (`llama3-8b-8192`) to extract:
   - Customer name
   - Email (handles spoken formats: "ram at gmail dot com", "r-a-m at the rate gmail.com")
   - Ticket subject (5–7 word title, NOT a raw sentence copy)
6. Form auto-fills: Subject, Customer Message, Name, Email
7. `source` field automatically set to `"voice"` → ticket appears under Voice filter
8. Confidence info shown: filename, char count, detected name/email/subject

### Manual Form Fields
- Subject, Customer Message, Name, Email, Customer Tier (Standard/Premium/Enterprise)
- **Quick Load** — 4 pre-built sample tickets (payment failure, churn threat, crash, export request)

### 🤖 8-Step AI Pipeline (animated in real-time)

| Step | Name | What happens |
|------|------|-------------|
| 1 | 🔒 PII Masking | Strips names, emails, phone/card numbers before sending to LLM |
| 2 | 😡 Emotion Detection | Classifies emotion type + intensity (0–10) |
| 3 | 🎯 Intent Classification | Tags intent: billing / refund / bug_report / cancellation / etc. |
| 4 | ⚠️ Churn Risk Analysis | Detects cancellation/competitor language → flags as churn risk |
| 5 | 📊 Urgency Scoring | 0–100 composite: emotion × tier × SLA × churn signals |
| 6 | 🏷️ Auto-Tagging (SLM) | Product area, platform, severity (P1/P2/P3) via `llama3.2:3b` |
| 7 | 📝 3-Line Summary (SLM) | Concise 3-point summary via `llama3.2:3b` |
| 8 | 🔍 RAG KB Search | ChromaDB vector search → top-3 KB article matches with confidence |

**Pipeline result panel shows:**
- Urgency level + score, Emotion, Intent
- Churn Risk banner, Human Required banner
- Severity, Product Area, Source
- 3-line AI summary
- Duplicate warning (if similar ticket exists)

### 🔁 Duplicate Ticket Detection
- Every new ticket is compared against all open tickets using vector similarity
- If similarity ≥ threshold → ticket flagged as `duplicate_of_id`
- Warning banner shown: "This ticket is **87% similar** to Ticket #42 — review before processing"
- Prevents agents working on the same issue twice

## 8️⃣ 📡 Data Sources & Ingestion

**URL:** `/sources`

### 🚨 Live Spike Alerter (top section)
- **APScheduler** background job runs every **5 minutes**
- Compares last-30-minute ticket volume vs 7-day hourly average **per product category**
- Thresholds:
  - `warning` → 1.5× average
  - `critical` → 2× average
- Alert cards show: category, current count vs baseline, severity badge
- Dismiss with one click
- Reappears if spike continues in next check cycle
- Alerts also shown on Manager Dashboard

### Ingestion Sources

#### 🟠 Reddit (PRAW)
- Configure subreddits (e.g. `r/techsupport`, `r/softwaregore`)
- Filters posts by complaint keywords
- Set limit per subreddit
- **Run Now** → fetches posts → runs them through full AI pipeline → appear in Agent Queue

#### 🍎 Apple App Store
- Fetches 1–2 star reviews via Apple RSS feed
- Enter Apple App ID (e.g. `1232780281` for Notion)
- No API key required
- Reviews processed as tickets automatically

#### 🎮 Google Play Store
- Fetches 1–2 star reviews via `google-play-scraper`
- Enter package name (e.g. `com.notion.id`)
- Reviews processed as tickets automatically

#### 📧 Email (Gmail / Outlook)
- IMAP connection — reads unread emails from configured mailbox
- Configured via `EMAIL_ADDRESS` + `EMAIL_APP_PASSWORD` in `.env`
- Converts each email → ticket through AI pipeline

#### 🎙️ Voice (Whisper)
- Shows the voice transcription module status
- Displays last transcribed audio info
- Uses OpenAI Whisper `base` model locally

### Source Status Panel
Each source shows:
- Last run time
- Status: `never / running / ok / error`
- Ticket count ingested
- **Run Now** button with configurable parameters
- 10-second live polling for status updates

## 🎯 End-to-End Demo Flow

1. **Login** → Sign in with Google or Microsoft (@relanto.ai)
2. **Submit voice ticket** → Upload MP3 → auto-transcribed → form filled → pipeline runs → ticket created
3. **Agent Queue** → Ticket appears sorted by urgency with all AI tags
4. **Open ticket** → Read summary → Customer 360 → Find Solution → Generate Draft → Send via mail client
5. **Spike demo** → Submit 5+ same-topic tickets → wait 5 min → spike alert fires on Dashboard + Data Sources
6. **Reddit ingestion** → `/sources` → enter subreddits → Run Now → tickets appear in queue
7. **Manager Q&A** → `/dashboard` → ask plain-English question → LLM answers with data
8. **Analytics** → `/analytics` → 5 tabs of deep visualization
9. **Weekly Digest** → `/digest` → Regenerate → AI narrative + recommended actions
